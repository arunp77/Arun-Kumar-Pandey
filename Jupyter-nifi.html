JupyterLab and Apache Nifi Configuration

Setting up Jupyter lab on EC2 machine

  • Once you have setup the Jupyterlab on the ec2 machine, you just need to run `docker-compose up -d` so as to start the Jupyter lab kernal (The `-d` flag runs the containers in detached mode, allowing them to run in the background).
  • Once the Docker Compose services are up and running, you can access JupyterLab in your web browser. By default, JupyterLab is often available at port 8888.
  • Open a web browser and navigate to: 
    http://[your-ec2-instance-ip]:4888/lab?
    You should be able to access JupyterLab.
  • When you access JupyterLab for the first time, it maybe asked to provide an authentication token or password. You can find this token in the logs or terminal output when you ran docker-compose up `-d`. Look for a line that starts with "token=" and contains a long string of characters. Copy and paste this token when prompted in your web browser.
  • Once you've authenticated, you can start using JupyterLab to run notebooks and perform data analysis or development tasks.

Setting up Apache nifi on EC2 machine

Using Apache NiFi to Transfer CSV Files to AWS S3
  • Step 1: Prepare Your EC2 Instance

    • Ensure that you have Apache NiFi installed and running on your EC2 instance. You can download and set up Apache NiFi by following the official documentation: NiFi Getting Started

  • Step 2: Configure NiFi for S3

    • 1. Start the NiFi UI by navigating to http://:2080/nifi.

    2. In the NiFi UI, you will design a flow to ingest data from your Jupyter Notebook to an S3 bucket.

    3. Create a processor group (a container for processors) in NiFi to organize your flow.

    4. Inside the processor group, add the following processors:

    • GetFile: Use this processor to fetch the generated CSV files from a local directory on your EC2 instance. Configure it to monitor the directory where your Jupyter Notebook saves the files.
    • PutS3Object: Use this processor to upload the files to an S3 bucket. Configure it with your AWS credentials and the target S3 bucket information.

    5. Connect the processors within your processor group to create the data flow. The flow should look like this:

                            GetFile -> PutS3Object
  • Step 3: Configure GetFile Processor

    • 1. Double-click on the GetFile processor and configure the properties:

    • Set the Input Directory to the directory where your Jupyter Notebook saves the CSV files.

    2. Configure the Scheduling tab to determine how often the processor checks for new files.

  • Step 4: Configure PutS3Object Processor

    • 1. Double-click on the PutS3Object processor and configure the properties:

    • Set the AWS Credentials Provider Service to an AWS credentials provider that has access to the S3 bucket.
    • Specify the Bucket to the S3 bucket where you want to upload the CSV files.
    • Set the Key to the desired path in the S3 bucket where you want to store the files.

    2. Configure the Scheduling tab to determine how often the processor should run.

  • Step 5: Start the Flow

    • Start the processor group to activate the flow. NiFi will begin monitoring the directory for new files and uploading them to the S3 bucket as they arrive.

  • Step 6: Monitor and Troubleshoot

    • You can monitor the status of the processors and view logs in the NiFi UI to ensure that the data flow is working correctly. Additionally, inspect the S3 bucket to confirm that the CSV files are being uploaded as expected.
